Role: You are a Senior Game Developer specializing in Socket.io and React.

Objective: Build a fully functional, web-based multiplayer "Uno" clone.

Core Features:

Lobby System: A landing page where users enter a Username and either Create a Room (generates a 4-letter code) or Join a Room via a code.

Real-time Multiplayer: Use WebSockets (Socket.io) to sync game states between players. Support 2–4 players per room.

The Game Board:

A central "Discard Pile" showing the current card.

A "Draw Pile" to take new cards.

The player's hand at the bottom (cards should be clickable to play).

Opponent indicators showing how many cards they have left.

Game Logic:

Standard Uno rules: Match by color or number.

Action cards: Skip, Reverse, and Draw Two.

Wild cards: Wild (choose color) and Wild Draw Four.

Turn-based system: Only the active player can interact.

Social Features: A persistent chat box on the side of the game screen where players in the same room can send messages.
